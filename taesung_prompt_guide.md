# 법인 재무관리시스템 - 프롬프트 가이드

## 🎯 이 문서의 목적
Claude AI를 사용하여 **일본 소규모법인**용 재무관리 웹앱을 처음부터 만들 수 있는 프롬프트 가이드입니다.

---

## 📋 사전 준비

### 필요한 것
- Claude Pro 계정 (코드 실행 가능)
- GitHub 계정 + 레포지토리
- Firebase 계정 (동기화용, 선택)
- 회사의 계정과목표 (일본 기준)
- 기존 전표 데이터 (Excel)

### 기술 스택
- **프론트엔드**: HTML + CSS + Vanilla JavaScript (5개 파일)
- **배포**: GitHub Pages (무료)
- **DB**: localStorage + Firebase Firestore (무료)
- **빌드 도구 불필요**: npm, webpack 등 없이 순수 HTML/JS

### 2-2. 재무제표 자동 연동
```
재무제표가 전표에서 100% 자동 집계되게 해주세요.

[핵심 원칙]
- 모든 BS/PL 항목은 acctBal() 함수로 전표에서 계산
- 유가증권평가손만 보유종목 시가 기준 자동반영
- 유가증권(BS)도 시가 조정 동시 반영
- 자산합계 = 부채+순자산합계 항상 일치 (DIFF=0)
- 새 계정으로 전표 치면 재무제표에 자동 표시

[동적 표시]
- 잔액 있는 모든 비용계정 → 판관비에 자동 표시
- 잔액 있는 모든 수익계정 → 영업외수익에 자동 표시
- 잔액 있는 모든 부채계정 → 대차대조표에 자동 표시
```

## ⚠️ 주의사항 (경험에서 얻은 교훈)

### 대차대조표 차대균형
- 재무제표는 **100% 전표 기반**으로 해야 차대가 맞음
- 유가증권만 시가 조정 (evalAdj = 시가평가손 - 전표평가손)
- BS 유가증권 = 장부가 - evalAdj → P/L과 동일 시가 기준

### localStorage 주의
- 기기별 독립 저장 (동기화 안 됨)
- data.js에서 D.xxx 참조 불가 (D는 app.js에서 생성)
- 초기화하면 시세 입력값도 리셋됨

### Firebase 동기화
- 자동 동기화하면 초기 데이터가 서버를 덮어쓸 수 있음
- **수동 업로드/다운로드가 안전**
- 다운로드 시 D = fb.data가 아닌 **속성별 복사** 필요

### 전표 데이터 검증
- 매각이익: 순이익 vs 총이익 주의
- 매수수수료: 취득원가에 포함 여부 확인
- 엑셀 원본과 계정별 잔액 대조 필수
